# 自研工具

Amagari Studio 不只处理翻译文本，也会围绕 Minecraft 本地化流程开发专用工具，把语言内容的接入、同步、校对和验证连接成可持续维护的工作流。目前公开介绍的第一个项目是 **Amagari Translation Tool**。

## Amagari Translation Tool

> 一个面向 Minecraft 地图与模组本地化的 Fabric 翻译辅助模组。它让语言内容可以直接进入游戏运行环境，并为多人协作、远程同步和双语校对提供统一支持。

**GitHub：** <https://github.com/zuiter/Amagari-Translation-Tool>

### 运行时语言加载

工具可以读取与 Minecraft 资源包语言文件兼容的 JSON，并把翻译内容动态加载到客户端翻译表。语言数据可以来自本地地图、远程服务器、局域网主机或 ParaTranz 项目，因此地图本地化不必反复制作和切换额外资源包。

它同时支持同一语言拆分为多个文件，并按照明确的优先级组合会话翻译、地图或远程语言、已启用资源包和原版语言表。这样既能保留原作资源，也能让地图专属文本获得稳定、可预测的覆盖结果。

### 远程与局域网同步

在多人服务器或局域网地图中，主机可以向加入者发布语言清单。客户端只获取本地缺失或内容哈希发生变化的数据，并通过压缩传输与本地缓存减少重复下载。

这套同步机制让翻译内容能够跟随服务器版本更新，也让参与测试的玩家保持一致的语言环境。缓存按服务器与语言隔离，并具备过期清理和历史版本数量控制，兼顾更新效率与本地空间占用。

### ParaTranz 协作连接

工具可以连接 ParaTranz 项目，把协作平台中的语言 JSON 拉取到当前游戏会话并立即应用，也可以在需要时写入当前本地地图语言文件。

这使译者能够从在线协作、项目导出一路走到游戏内验证，减少手动下载、重命名、复制和重新打包造成的流程中断。项目访问、导出结果与本地缓存都由工具统一管理，适合持续迭代的地图汉化任务。

### 双语校对与源文对照

工具提供目标语言与源语言切换，并能在游戏内显示可追溯的源文。当前覆盖的校对场景包括物品提示、带翻译条目的告示牌以及书本文本。

源文显示会尽量保留原组件的颜色、斜体、参数和排版信息，使译者可以在真实游戏语境中比较译文与原文，而不是只在表格中孤立地检查字符串。对于大厅公告、任务提示和规则说明等固定告示牌内容，工具也能通过专用翻译条目完成客户端显示层面的本地化，不修改地图存档本身。

### 工程可靠性

- 使用语言清单、内容哈希和压缩数据完成按需同步。
- 区分本地地图、服务器和不同语言的缓存，避免数据互相污染。
- 自动清理长期未使用的远程缓存，并限制同一服务器语言版本的保留数量。
- 仅向命令执行者返回私有反馈，避免多人服务器中的操作信息干扰其他玩家。
- 对单次远程语言数据设置体积边界，鼓励地图项目保留必要的语言和命名空间。
- 保持语言覆盖顺序明确，便于定位资源包、地图语言与在线协作结果之间的冲突。
